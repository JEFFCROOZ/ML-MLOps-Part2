## Hero Copy

**Headline:**
You built it. Now let's keep it running.

**Subheadline:**
If you made it through Part 1, you know how to build a recommendation engine. You trained a model, evaluated it, and saw it work. That's a real accomplishment. But here's the thing nobody tells you right away: building the model is only half the job.

The other half is everything that happens after.

This is Part 2. We're going to talk about what it actually takes to maintain a machine learning system — how to track your experiments, version your models, keep your data fresh, know when your model is breaking down, and build the kind of infrastructure that makes ML sustainable over time.

No new tools required. No cloud accounts needed. Just the concepts, the patterns, and the reasoning behind them — so that when you do sit down to build this for real, you'll know exactly what you're building toward.

## A Note on Tools

You'll see real tools mentioned throughout this course — tools like MLflow for experiment tracking, Airflow for pipelines, and others. We name them because they're concrete handles for abstract ideas, not because they're the only options.

The concepts here apply regardless of your stack. If you're running on cloud infrastructure, on-prem, or a laptop — the operational thinking is the same.

---

## Key Takeaway

A model that nobody maintains is a model that slowly stops being right. Part 2 is about building the habits, systems, and instincts that keep yours accurate, reliable, and improving over time.
